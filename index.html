Mysql-索引

发表于 2016-08-28   |   分类于 linux   |  

索引类型

索引是在MYSQL的存储引擎层中实现的，而不是在服务层实现的。所以每种存储引擎的索引都不一定完全相同，也不是所有的存储引擎都支持所有的索引类型。MYSQL目前提供了一下4种索引。
    ○ B-Tree 索引：最常见的索引类型，大部分引擎都支持B树索引。
    ○ HASH 索引：只有Memory引擎支持，使用场景简单。
    ○ R-Tree 索引(空间索引)：空间索引是MyISAM的一种特殊索引类型，主要用于地理空间数据类型。
    ○ Full-text (全文索引)：全文索引也是MyISAM的一种特殊索引类型，主要用于全文索引，InnoDB从MYSQL5.6版本提供对全文索引的支持。

B-TREE
    ○ 普通索引
    新增-create index index_name on table(column(prefix_length)…)
    修改-alter table table_name add index index_name (column(prefix_length)…)
    删除-drop index  index_name on table_name;  alter table table_name drop index_name
    查看-show index from tbl_name, show keys from tbl_name

    ○ UNIQUE索引
    唯一性(不允许重复),如身份证索引
    ○ 主键：PRIMARY KEY索引
    主键是一种唯一性索引，但它必须指定为“PRIMARY KEY”
    一个表只可能有一个PRIMARY KEY索引，因此不需要指定索引名

Linux 常用命令

发表于 2016-08-28   |   分类于 linux   |  

进程控制

1. Mysql
    检测是否安装Mysql输入：     #rpm -qa | grep -i mysql
    查看mysql运行状态    ps -ef|grep mysqld
    强制关闭MySQL    killall mysql
    停止    /usr/bin/mysqladmin -u root -p shutdown
    启动    /etc/init.d/mysql start
2. Redis
    检查Redis服务器系统进程    ps -aux|grep redis例子：
        redis     4162  0.1  0.0  10676  1420 ?        Ss   23:24   0:00 /usr/bin/redis-server /etc/redis/redis.conf
        conan     4172  0.0  0.0  11064   924 pts/0    S+   23:26   0:00 grep --color=auto redis
    通过启动命令检查Redis服务器状态    netstat -nlp|grep 6379；sudo netstat -tap | grep mysql
    通过启动命令检查Redis服务器状态    sudo /etc/init.d/redis-server status

3. 查看某一端口的占用情况    lsof -i:端口号     
4. 解压    unzip -o cms.war -d cms    
5. 安装记录    dpkg-query -l 'foo*'    
6. 查看系统    lsb_release -a; uname -a
7. 后台运行     nohup … >out.log &
